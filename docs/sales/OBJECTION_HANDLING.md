# Работа с возражениями

| Возражение | Честный ответ | Следующий шаг |
| --- | --- | --- |
| «Это готовый маркетплейс?» | Это commerce-платформа с seller tooling; полноценный multi-vendor settlement не входит в текущий scope. | Уточнить роли продавцов и расчеты |
| «Поддерживается эквайринг?» | Сейчас реализована ручная СБП с seller confirmation и 30-минутным сроком. | Оценить PSP/acquiring интеграцию |
| «Можно запуститься завтра?» | Только после production config, UAT, backup/monitoring и legal/security sign-off. | Составить launch plan и владельцев |
| «Все безопасно?» | Есть HMAC, RBAC, rate limits, audit, transactions и backup; независимый pentest еще является gate. | Согласовать security acceptance |
| «Возвраты соответствуют закону?» | Реализована 24-часовая механика, но применимость должен подтвердить юрист; gap уже зафиксирован. | Legal review и синхронизация кода/текста |
| «Можно скрыть товар?» | `is_listed=false` убирает его из discovery, но активная карточка доступна по прямой ссылке. | Если нужна секретность — отдельное требование |
| «Есть гарантированный SLA?» | SLA определяется договором и операционной конфигурацией, код сам его не гарантирует. | Оценить hosting/on-call/monitoring |
| «Легко интегрировать ERP?» | API модульный, но конкретной готовой ERP-интеграции не заявлено. | Discovery API/data mapping |
| «Telegram write access = реклама?» | Нет. Он разрешает service delivery и не включает marketing автоматически. | Настроить отдельный opt-in |
