# Приватность и персональные данные

StyleXac обрабатывает данные для идентификации в Telegram, оформления и доставки заказа, оплаты, возврата, поддержки, безопасности, аналитики и — только при отдельном допустимом основании — маркетинга.

## Принципы

- собирать только данные, необходимые конкретному процессу;
- не считать write access согласием на маркетинг;
- хранить подтверждаемую версию согласия/документа, если это требуется законом;
- ограничивать seller доступ собственным магазином и рабочей ролью;
- не логировать raw Telegram `initData`, секреты и полные чувствительные payloads;
- определить сроки хранения и безопасное удаление для БД, uploads, логов и backup;
- поддерживать запросы на доступ, исправление, удаление/ограничение и отзыв согласия в применимом объеме;
- документировать processors, трансграничные передачи и места хранения.

Текущая реализация содержит необходимые доменные данные, audit/analytics механизмы и управление подпиской, но готовность по 152-ФЗ, GDPR или иной применимой норме не заявляется. Карта обработки находится в [Personal Data Processing Map](../legal/PERSONAL_DATA_PROCESSING_MAP.md); список обязательных решений — в [Legal Readiness Checklist](../legal/LEGAL_READINESS_CHECKLIST.md).
