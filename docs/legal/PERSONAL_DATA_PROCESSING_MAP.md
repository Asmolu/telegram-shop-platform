# Карта обработки персональных данных

**Срез реализации:** `325e3af`, 13 июля 2026 года. **Статус:** основа для legal/privacy review, не реестр, утвержденный юристом.

| Процесс | Данные | Цель | Основные получатели/хранилища | Решение до запуска |
| --- | --- | --- | --- | --- |
| Telegram auth | user id, username/name, auth metadata; raw `initData` только на проверке | идентификация | backend, PostgreSQL для профиля | основание, notice, TTL auth logs |
| Профиль/checkout | телефон, имя, адрес, рост, вес | заказ и доставка | PostgreSQL, seller в scope заказа | обязательность полей, retention |
| Заказ | товары-снимки, суммы, статусы, контакты | исполнение сделки, учет | PostgreSQL, seller/admin | сроки и доступ |
| Ручная СБП | статус, инструкции, подтверждение seller; возможно evidence | оплата и сверка | PostgreSQL/uploads, seller/admin | ККТ, минимизация, deletion |
| Возврат | причина, комментарии, 1–5 фото/видео | рассмотрение возврата | PostgreSQL и файловое хранилище | чувствительность, malware scan, TTL |
| Уведомления | Telegram ids/chat id, opt-in, write access, delivery state | сервисные/маркетинговые сообщения | PostgreSQL, Telegram, Bot 1 | раздельные основания и unsubscribe |
| Поддержка | обращение, контакты, заказ | решение проблемы | support personnel, logs/DB по процессу | SLA, access, retention |
| Analytics/Audit | события поведения, actor/object, технические метаданные | продуктовая аналитика, безопасность | PostgreSQL и настроенная telemetry | перечень полей, pseudonymization, TTL |
| Backup | копия доменных данных | восстановление | локальное и удаленное хранилище | шифрование, доступ, удаление по циклу |

Нужно отдельно инвентаризировать фактические внешние processors, страны/дата-центры, договоры обработки, Telegram-роль, Caddy/hosting logs и все telemetry endpoints. Запрос на удаление должен учитывать юридически обязательное хранение заказов и инерцию backup; пользователю сообщается реальный процесс, а не мгновенное обещание.
