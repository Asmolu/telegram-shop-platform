# Gap-анализ политики возвратов

## Что реализовано

Canonical rule: [Returns and Refunds](../product/RETURNS_AND_REFUNDS.md).

Возврат создается для доставленного заказа в течение 24 часов от `delivered_at`, не более одного на заказ, может быть частичным и требует от 1 до 5 вложений. Разрешены JPEG, PNG, WebP, MP4, WebM и QuickTime, до 20 MiB на файл. Статусы: `PENDING → APPROVED | REJECTED | CANCELLED`, затем `APPROVED → COMPLETED | CANCELLED`. Возмещение создается вручную, доставка автоматически не входит в сумму; повторное пополнение остатка защищено delta-safe логикой.

## Юридические и продуктовые разрывы

| Разрыв | Риск | Требуемое решение |
| --- | --- | --- |
| Жесткое окно 24 часа | Может ограничивать обязательные права потребителя | Юрист определяет законные сроки; код и тексты синхронизируются |
| Одно заявление на заказ | Не покрывает последовательные дефекты/части заказа | Определить допустимую модель кейсов |
| Обязательные медиа | Может быть неправомерным барьером | Разрешить альтернативное доказательство/обращение поддержки |
| Доставка исключена по умолчанию | В некоторых случаях должна возмещаться | Формализовать причины и расчет доставки |
| Ручное возмещение | Риск задержки и ошибки | SLA, four-eyes, reconciliation, audit evidence |
| Нет явного exchange flow | Обмен обрабатывается вне домена возврата | Решить: отдельный процесс или документированный возврат+новый заказ |
| Нет customer cancel endpoint | Отмена до оплаты/исполнения не является самостоятельным UX | Определить права и реализовать согласованный процесс |
| Вложения | Privacy/malware/retention риск | Quarantine, access control, TTL и процедура удаления |

До юридического sign-off UI и продажи не должны обещать, что текущая 24-часовая механика исчерпывает права покупателя. Поддержка обязана иметь внепрограммный маршрут эскалации для законных требований.
