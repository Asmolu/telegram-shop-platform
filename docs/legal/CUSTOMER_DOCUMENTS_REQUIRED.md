# Документы для покупателя

Ниже — пакет, который должен подготовить и утвердить владелец бизнеса с юристом. Код не генерирует юридическую действительность сам по себе.

| Документ | Что должен раскрывать | Где предъявлять |
| --- | --- | --- |
| Публичная оферта / условия продажи | стороны, товар, цена, заказ, оплата, доставка, приемка, возврат, ответственность, споры | до checkout и по постоянной ссылке |
| Политика конфиденциальности | оператор, данные, цели, основания, сроки, получатели, права субъекта, контакты | до/при первом сборе данных |
| Согласие/уведомление на ПД | конкретные цели и действия, когда согласие является основанием | в соответствующей точке сбора |
| Политика доставки | методы, тарифы, регионы, сроки, риски и контакты | каталог/checkout/order |
| Политика возврата и возмещения | законные права, исключения, доказательства, сроки решения и выплаты | до покупки и в заказе |
| Правила оплаты СБП и чеков | реквизиты получателя, срок 30 минут, проверка, чек/возврат | payment instructions |
| Маркетинговое согласие | канал, содержание, частота, отзыв | отдельный opt-in, не write access |
| Правила отзывов и контента | право публикации, модерация, запреты, удаление | перед отправкой отзыва |
| Контакты и реквизиты | продавец/оператор, адрес, support, претензии | footer/profile/help |

Каждый документ получает версию, дату, владельца, применимую территорию и журнал изменений. UI должен хранить или передавать подтверждаемую версию принятого текста там, где это юридически необходимо.
